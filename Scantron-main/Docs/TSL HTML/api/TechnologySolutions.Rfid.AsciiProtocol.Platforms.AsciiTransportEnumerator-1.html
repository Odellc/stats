EnumerateExisting()

When overridden in a derived class. This should call Added(TTransport) or Removed(String, Boolean) to update Transports with the list of existing transports. During enumeration the state should be Started if the state is set external to Stopping the method should return.

Declaration
protected abstract Task EnumerateExisting()
Returns
Type Description
Task

The task to enumerate the existing transports
Remarks

The State property should be checked during long running tasks. If it is Stopping then the enumeration should be abandoned and return from the method call. StopWatching(Boolean) will then be called to fully stop the enumeration.

Exceptions can (and should) be thrown from this method for errors that prevent enumeration. This will cause a call to Abort(Exception) to transition the Enumerator to the Aborted state and the developer debugging the app can determine the issue (e.g. permission required)

Parameters
Type Name Description
Boolean resumed

Returns
Type Description
Task

The task that starts watching for changes to transports
Remarks

There is nothing to do if the current state is Started or EnumerationCompleted.

If the State is Stopping then it waits up to 2 seconds to reach Stopped

Assuming the state is Aborted, Created or Stopped the enumerator is then started with StartWatching(Boolean) and changes the state to Started. Then EnumerateExisting() is awaited to complete enumeration of existing devices. When the enumerate completes if the State is still Started it is set to EnumerationCompleted.
