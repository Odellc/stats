Getting Started - Commanding

Overview

Using a TSL ASCII 2 Reader from the API is all about sending commands. The commands are documented in Technology Solutions UK Ltd ASCII 2 Protocol

A command can be sent to the reader and it performs the command (send only)

A command can be sent to the reader and the response is captured (send and receive)

A trigger press (single or double click) can execute a command on the reader. Depending on the command it may repeat while the trigger is held.

Reader Operations

To perform most common operations we have introduced a simple high level API to achieve results quickly with the API. This API defines a number of IReaderOperation that can be linked to the readers trigger or run on demand. For more information see Getting Started - Reader Operations

ASCII Commander

For full control over the reader, which requires more knowledge of the reader, there is the AsciiCommander. The ASCII Commander when it has its responder chain configured can receive responses from all ASCII Protocol commands as well as send them. For more information see Getting Started - AsciiCommander
